# 本番環境更新手順

## 管理パスワード

投句フォーム以外の画面は、管理パスワードが必要。

初期値は

```
hisui
```

ログイン後、トップページの

管理パスワード変更

から変更できる。

本番公開前に変更すること。

必要に応じて、初期パスワードは環境変数

```
HISUI_ADMIN_PASSWORD
```

で変更できる。

セッション保護用に

```
HISUI_SECRET_KEY
```

も設定できる。

画面からパスワードを変更した後は、変更後のパスワードが優先される。

---

## 通常の更新手順

### 1. ローカルで動作確認

ローカル環境で

```
python app.py
```

を実行し、

追加・修正した機能が正常に動くことを確認する。

---

### 2. GitHubへPush

GitHub Desktopで

Commit

↓

Push origin

を実行する。

---

### 3. PythonAnywhereへログイン

https://www.pythonanywhere.com/

へログイン。

---

### 4. Bashコンソールを開く

Consoles

↓

Bash

---

### 5. リポジトリへ移動

```bash
cd ~/HisuiKukai
```

---

### 6. GitHubから最新版を取得

```bash
git pull
```

成功すると

```
Already up to date.
```

または

```
Updating xxxx...
```

と表示される。

---

### 7. バージョン確認（推奨）

最新版が反映されているか確認する。

```bash
git log --oneline -5
```

最新のCommit名が

```
v1.0.2 ○○
```

などになっていればOK。

---

### 8. マイグレーション実行

データベース構造を変更した場合のみ。

```bash
python migrate.py
```

成功例

```
既に存在します...
マイグレーションが完了しました。
```

---

### 9. 初回のみ（DBが無い場合）

```bash
python init_db.py
```

通常運用では不要。

※既存データがある場合は実行しない。

---

### 10. WebアプリをReload

Web

↓

Reload

を押す。

これを忘れると更新されない。

---

### 11. 動作確認

ブラウザで

https://nes710.pythonanywhere.com

を開き、

追加した機能が反映されているか確認する。

---
